36 # Configuration and Connection
37 
38 ## Configure Your Board
39 
40 When you use with UNO board, uncomment the follow line in `Eth_ESP8266.h`.
41 
42  #define UNO
43 
44 When you use with MEGA board or WBoard, uncomment the follow line in `Eth_ESP8266.h`.
45 
46  #define MEGA
47 
48 ## Connection
49 
50 ### UNO
51 
52 When you use it with UNO board, the connection should be like these:
53 
54  ESP8266_TX->D0
55 
56  ESP8266_RX->D1
57 
58  ESP8266_CHPD->3.3V
59 
60  ESP8266_VCC->3.3V
61 
62  ESP8266_GND->GND
63 
64  //The baud rate of software serial can't be higher that 19200, so we use software serial as a debug port
65  FTDI_RX->D3
66 
67  FTDI_TX->D2
68 
69 ### MEGA
70 
71 When you use it with MEGA board, the connection should be like these:
72 
73  ESP8266_TX->RX1(D19)
74 
75  ESP8266_RX->TX1(D18)
76 
77  ESP8266_CH_PD->3.3V
78 
79  ESP8266_VCC->3.3V
80 
81  ESP8266_GND->GND
82 
83 When you want to output the debug information, please use DebugSerial. For example,
84 
85  DebugSerial.println("hello");
86 
87 ### WBoard
88 
89 Need no connections.

91 # Attention
92 
93 **Note1**: The size of message from ESP8266 is too big for arduino sometimes,
94 so the library can't receive the whole buffer because the size of the hardware
95 serial buffer which is defined in HardwareSerial.h is too small.
96 
97 Open the file from `\arduino\hardware\arduino\avr\cores\arduino\HardwareSerial.h`.
98 See the follow line in the HardwareSerial.h file.
99 
100  #define SERIAL_BUFFER_SIZE 64
101 
102 The default size of the buffer is 64. Change it into a bigger number, like 256
103 or more.
104 
105 The SRAM size of mega is bigger than UNO's, so it is better to use MEGA board to
106 communicate with ESP8266.
